# 开发规范

本文件是 `$python-requests` 的执行清单。如果本文件与 `SKILL.md` 冲突，以 `SKILL.md` 为准。

## 抓包前

- 确认目标页面、URL、操作路径和成功标准
- 只接管已打开的 Chrome `127.0.0.1:9222`
- 先检查 `http://127.0.0.1:9222/json` 或 `http://127.0.0.1:9222/json/version`
- 如果当前页面工具只看到新的 `about:blank`、空白页或孤立受管会话，立即停止该路径并改走直接 CDP 附着
- 禁止新开浏览器或新起 Chrome 实例
- 禁止通过 `new_page`、重新导航空白页或重新登录站点来“补救”附着失败
- 围绕用户具体操作抓包，不做无关流量泛抓
- 9222、`/json`、`/json/version`、`webSocketDebuggerUrl` 仅用于抓包分析，不得写入最终交付脚本

## 请求链

- 找出初始化、鉴权、分页、详情、导出、下载链
- 区分业务接口与埋点、静态资源、日志、`OPTIONS`
- 记录每个后续请求依赖的前置 ID、credential、下载 URL

## 认证链

- 禁止通过 `Runtime.evaluate`、JS 注入、`localStorage`、`sessionStorage`、`document.cookie`、页面全局变量或直接调用前端函数，提取现成 `access_token`、cookie、签名结果作为最终方案
- 认证优先级固定为：
  1. 还原登录、刷新、换票、前置 credential、签名链
  2. 纳入前置接口返回
  3. 用户显式传入
  4. 用户明确允许时再用环境变量
- cookie 默认由用户以字符串形式传入，不自动从浏览器读取
- 如果认证链无法安全重放，就退化为用户输入，不要自动抽取

## 落表

- 所有导出表头必须以前台显示表头为准
- 表头顺序必须与页面显示顺序一致
- 只有在脚本需要把接口响应重组为表格导出时，才额外生成 `field_mapping.json`
- 如果脚本只是复刻原始下载请求并保存服务端直接返回的文件，则不生成 `field_mapping.json`
- 表格导出场景下，运行时优先直接读取 `field_mapping.json`，不要在 `.py` 内重复维护同一份映射
- 派生字段使用 `__derived__` 标记来源

## 脚本形态

- 主脚本默认命名为英文 `snake_case`，例如 `platform_section_function.py`
- 如果生成目录，目录名默认也命名为英文 `snake_case`
- 中文业务词翻译成英文语义词，不直接使用中文文件名
- 默认至少包含 1 个清晰的业务 `class`
- 每个函数和每个方法上方都写一句简短注释
- 遵守高内聚、低耦合原则，一个功能只在一个方法内完成，禁止抽离无关的小方法
- 最终脚本禁止包含 `cdp_debug_url`、`webSocketDebuggerUrl`、`127.0.0.1:9222`、`/json`、`/json/version` 等 CDP 接管参数或检查逻辑
- `self.xxx` 只保存跨多个方法复用的长期状态，例如 `session`、字段映射、稳定配置
- 某次方法调用才会使用的输入值，直接作为方法参数或局部变量传递，不要机械地全部挂到 `self`
- `csv_path`、`mapping_output_path`、`json_path`、临时筛选值、分页值这类当前一次调用就能传完的参数，默认不要挂到 `self`
- 请求头默认写成常量，类内通过 `update_headers()` 之类的方法统一更新会话头
- 避免生成 `main.py`、`utils.py`、通用 client、插件式抽象

## 函数规模

- 仅保留 `fetch_*`、`build_*`、`write_*`、`run()`、`main()`
- 如有需要，可额外保留一个 `update_headers()` 方法来封装会话头更新
- 一般控制在 1 个 `class` 和少量主要方法
- 能并入现有方法的逻辑，不再额外拆 helper
- 只有真实复用、独立签名/加密链路或清晰业务边界，才允许拆出新方法

## 编码

- 使用 `requests.Session`
- 每个请求显式设置 `timeout`
- 使用 `raise_for_status()`，并检查接口自己的 `success/code/msg`
- 优先标准库 `csv`、`json`、`Path`
- 不把表头直接写成 response 字段名

## 验证

- 核对脚本输出与页面结果：数量、ID、日期范围、样例行、表头顺序；表格导出再核对字段映射 JSON，原始下载再核对文件名、后缀、大小或样例打开结果
- 说明哪些认证参数是脚本生成、哪些来自前置接口、哪些由用户传入
- 确认最终脚本已经移除抓包阶段使用的 CDP、9222 和页面接管检查逻辑
- 确认没有把受管新浏览器或空白页误判成用户当前页面
- 确认对象属性没有滥用为一次性调用参数
- 确认输出路径等一次性参数没有被错误保存到对象属性
- 写明跳过的端点和已知限制
