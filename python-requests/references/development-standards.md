# 开发规范

本文件是 `$python-requests` 的执行清单。如果本文件与 `SKILL.md` 冲突，以 `SKILL.md` 为准。

## 抓包前

- 确认目标页面、URL、操作路径和成功标准
- 先判断任务是复用当前已登录页面，还是新建受控浏览器会话
- 复用当前已登录页面时，只接管已打开的 Chrome `127.0.0.1:9222`
- 先检查 `http://127.0.0.1:9222/json` 或 `http://127.0.0.1:9222/json/version`
- 如果当前页面工具只看到新的 `about:blank`、空白页或孤立受管会话，立即停止该路径并改走直接 CDP 附着
- 禁止新开浏览器或新起 Chrome 实例
- 禁止通过 `new_page`、重新导航空白页或重新登录站点来“补救”附着失败
- 新建受控浏览器会话时，优先尝试 CloakBrowser，其次尝试 Scrapling，最后再退回普通 CDP/浏览器路径
- CloakBrowser 只有在能暴露用户指定 profile、目标标签页、可验证登录态，或用户明确要求新建受控会话时才使用
- Scrapling 适合无须复用当前真实标签页的抓取、动态渲染和用户显式传入认证参数的场景，不得把它声称为当前页面抓包
- 接管入口用于定位接口，必须能执行用户页面操作，并输出请求方法、URL、query、body、headers、响应状态、响应体或下载链
- 如果 CloakBrowser、Scrapling 或 CDP 只能渲染页面但不能可靠输出请求链，只能作为观察辅助，不能作为接口定位主入口
- CloakBrowser、Scrapling、CDP/9222 都只属于分析入口，最终交付脚本不得包含这些接管逻辑
- 围绕用户具体操作抓包，不做无关流量泛抓
- 9222、`/json`、`/json/version`、`webSocketDebuggerUrl` 仅用于抓包分析，不得写入最终交付脚本

## 请求链

- 找出初始化、鉴权、分页、详情、导出、下载链
- 接管执行操作前先开启网络捕获，操作完成后再停止捕获并筛选业务请求
- 区分业务接口与埋点、静态资源、日志、`OPTIONS`
- 记录每个后续请求依赖的前置 ID、credential、下载 URL

## 重试与异常

- 只对网络超时、连接断开、`408`、`429`、`500`、`502`、`503`、`504` 这类临时错误做有限重试
- 重试必须设置最大次数、明确 `timeout` 和退避等待，默认不超过 3 次
- 可能创建任务或触发导出的 POST 请求，只有具备幂等键、可查重任务 ID 或先查询已有任务时才自动重试
- `401`、`403`、未登录、未选身份、无权限、身份失效、参数错误、验证码、MFA、风控等确定性错误直接抛出
- “排队中、生成中、处理中”按异步轮询处理，必须设置最大等待时间
- 接口成功但数据为空时，不默认报错；页面或汇总接口显示有数据但明细为空时才抛数据一致性异常
- 异常信息必须包含子任务名、请求方法、URL、关键参数、HTTP 状态码、接口 `code/msg/success` 和响应摘要

## 大任务状态

- 一个大采集任务包含多个报表、店铺、日期分片或子采集脚本时，必须记录任务状态 JSON
- 每个子任务使用稳定 `task_key`，并记录 `input_hash` 防止参数变化后误跳过
- `success` 才算已完成；下次重跑时 `task_key + input_hash` 一致且输出存在，自动跳过
- `failed` 要记录错误类型、错误信息、失败时间、是否可重试；当前大任务继续执行其他不依赖它的子任务
- `running`、`partial`、`failed`、`blocked` 都不算已完成，下次重跑默认重新尝试
- 全局认证、身份、权限或风控错误应停止整个大任务，不继续生成大量无效失败记录
- 状态文件默认命名为 `task_state.json`，作为运行参数传入 `run()` 或 `main()`，不要默认挂到 `self`
- 状态结构可参考 `references/task-state-template.json`

## 认证链

- 禁止通过 `Runtime.evaluate`、JS 注入、`localStorage`、`sessionStorage`、`document.cookie`、页面全局变量或直接调用前端函数，提取现成 `access_token`、cookie、签名结果作为最终方案
- 认证优先级固定为：
  1. 还原登录、刷新、换票、前置 credential、签名链
  2. 纳入前置接口返回
  3. 用户显式传入
  4. 用户明确允许时再用环境变量
- cookie 默认由用户以字符串形式传入，不自动从浏览器读取
- 如果认证链无法安全重放，就退化为用户输入，不要自动抽取

## 落表

- 所有导出表头必须以前台显示表头为准
- 表头顺序必须与页面显示顺序一致
- 只有在脚本需要把接口响应重组为表格导出时，才额外生成 `field_mapping.json`
- 如果脚本只是复刻原始下载请求并保存服务端直接返回的文件，则不生成 `field_mapping.json`
- 表格导出场景下，运行时优先直接读取 `field_mapping.json`，不要在 `.py` 内重复维护同一份映射
- 派生字段使用 `__derived__` 标记来源

## 输出文件

- 生成表格文件名按 `数据抓取时间-业务日期-业务导出表名称.<suffix>` 生成，例如 `20260616235435-20260615-已购客核心指标.csv`
- 文件名不要包含店铺 ID、店铺名、商品 ID、SKU、商品名称、直播间名称、直播场次 ID、账户名、账户 ID 等业务维度；这些关键字段必须写入文件内容
- 表格内容优先补齐数据日期、店铺ID、店铺名、商品ID、SKU、商品名称、直播间名称、直播场次ID、账户名、账户ID、文件名；当前业务没有对应维度时可留空
- 爬取接口数据并重组成表格时，最终输出固定为 `.csv`，使用 `encoding="utf-8"` 和 `newline=""`
- 非系统自生成的文本、CSV、JSON、字段映射、任务状态和脚本文件统一使用 UTF-8 编码；服务端原始下载文件按响应二进制原样保存，不强行转码
- 下载报表或二进制文件时，保留服务端原始后缀，只规范文件名前缀
- 同一脚本包含多个采集请求任务、多个报表或多个业务导出表时，按请求任务或业务导出表名称拆分多个输出文件，不默认合并
- 多子任务状态里的 `output_file` 记录实际输出文件路径，跳过已完成任务时要同时确认文件存在

## 脚本形态

- 主脚本默认命名为英文 `snake_case`，例如 `platform_section_function.py`
- 如果生成目录，目录名默认也命名为英文 `snake_case`
- 中文业务词翻译成英文语义词，不直接使用中文文件名
- 默认至少包含 1 个清晰的业务 `class`
- 每个函数和每个方法上方都写一句简短注释
- 对应页面操作、接口请求、导出链路或子任务编排的类/方法注释必须包含“操作步骤”和“访问网址”；“操作步骤”写页面上点击哪里、选择哪里、如何筛选或导出才能触发该请求，用 `-` 分割；“访问网址”写浏览器地址栏页面 URL，不写接口 path 或 API URL
- 没有外部访问网址的本地处理方法只写职责，不强行补 URL
- 采集请求方法体开头必须用三引号写参数注释，说明参数对应功能、类型、是否必填、格式或来源
- 实际 `query`、`json`、`form data`、分页参数构造前也要用三引号写请求参数注释，避免只看 payload 时不知道字段含义
- 遵守高内聚、低耦合原则，一个功能只在一个方法内完成，禁止抽离无关的小方法
- 最终脚本禁止包含 `cdp_debug_url`、`webSocketDebuggerUrl`、`127.0.0.1:9222`、`/json`、`/json/version` 等 CDP 接管参数或检查逻辑
- `self.xxx` 只保存跨多个方法复用的长期状态，例如 `session`、字段映射、稳定配置
- 某次方法调用才会使用的输入值，直接作为方法参数或局部变量传递，不要机械地全部挂到 `self`
- `csv_path`、`mapping_output_path`、`json_path`、临时筛选值、分页值这类当前一次调用就能传完的参数，默认不要挂到 `self`
- 请求头默认写成常量，类内通过 `update_headers()` 之类的方法统一更新会话头
- 避免生成 `main.py`、`utils.py`、通用 client、插件式抽象

## 函数规模

- 仅保留 `fetch_*`、`build_*`、`write_*`、`run()`、`main()`
- 如有需要，可额外保留一个 `update_headers()` 方法来封装会话头更新
- 一般控制在 1 个 `class` 和少量主要方法
- 能并入现有方法的逻辑，不再额外拆 helper
- 只有真实复用、独立签名/加密链路或清晰业务边界，才允许拆出新方法

## 编码

- 使用 `requests.Session`
- 每个请求显式设置 `timeout`
- 使用 `raise_for_status()`，并检查接口自己的 `success/code/msg`
- 生成有限重试逻辑和明确异常分类，不吞异常、不无限重试
- 多子任务采集时生成断点重跑逻辑，自动跳过已完成子任务，失败子任务记录后继续
- 优先标准库 `csv`、`json`、`Path`
- 不把表头直接写成 response 字段名
- 检查相关类/方法注释是否能直接对应到页面点击路径和浏览器地址栏页面 URL，不能把接口 path 当成访问网址
- 检查采集请求方法和请求 payload 是否有三引号参数注释

## 验证

- 核对脚本输出与页面结果：数量、ID、日期范围、样例行、表头顺序；表格导出再核对字段映射 JSON、UTF-8 CSV 编码、命名格式和统一关键字段是否落入文件内容，原始下载再核对文件名、原始后缀、大小或样例打开结果
- 说明哪些认证参数是脚本生成、哪些来自前置接口、哪些由用户传入
- 核对重试只覆盖临时错误，认证/身份/权限/参数错误会直接抛出
- 多子任务场景核对 `task_state.json` 中已完成、跳过、失败、未完成数量是否准确
- 确认最终脚本已经移除抓包阶段使用的 CDP、9222 和页面接管检查逻辑
- 确认没有把受管新浏览器或空白页误判成用户当前页面
- 确认对象属性没有滥用为一次性调用参数
- 确认输出路径等一次性参数没有被错误保存到对象属性
- 写明跳过的端点和已知限制
