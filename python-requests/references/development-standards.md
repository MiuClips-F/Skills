# 开发规范

本文件是 `$python-requests` 的执行清单。如果与 `SKILL.md` 冲突，以 `SKILL.md` 为准。

## 抓包前

- 明确目标页面、页面 URL、操作路径、筛选条件、时间范围、导出格式和成功标准。
- 复用当前已登录页面时，优先用 chrome-devtools-mcp 匹配用户已打开的 Chrome 标签页。
- 如果 MCP 只看到新的 `about:blank`、空白页、孤立受管会话，或必须 `new_page` 才能继续，立即停止该路径并改走直接 CDP。
- 只有 MCP 路径失败或请求元数据不足时，才检查 `http://127.0.0.1:9222/json` 或 `/json/version`。
- 禁止新开浏览器、新起 Chrome、重新导航空白页或重新登录站点来伪造当前页面。
- CloakBrowser/Scrapling 只在用户明确要求新建受控会话，或能暴露可验证的既有 profile/标签页时使用。
- 所有接管工具只属于分析入口；最终脚本不得包含 MCP/CDP/9222/HAR/页面自动化逻辑。

## 请求链

- 抓包时先开启捕获，再执行用户页面操作，操作结束后停止捕获并筛选业务请求。
- 区分初始化、鉴权、刷新、业务、分页、详情、导出、轮询、下载和噪声请求。
- 记录每个后续请求依赖的前置 ID、credential、token、签名字段、下载 URL。
- 忽略埋点、日志、静态资源和普通 `OPTIONS`，除非它们揭示必要请求头。

## 认证链

- 禁止通过 `Runtime.evaluate`、JS 注入、`localStorage`、`sessionStorage`、`document.cookie`、页面全局变量或前端函数调用，提取现成 token/cookie/签名结果作为最终方案。
- 认证优先级：还原登录/刷新/换票/签名链 -> 纳入前置接口返回 -> 用户显式传入 -> 用户明确允许时再用环境变量。
- cookie 默认由用户以字符串形式传入。
- 找不到可重放认证链时，退化为用户输入，并在最终说明中写明限制。

## 脚本形态

- 主脚本默认英文 `snake_case` 命名，不生成 `main.py`。
- 默认至少 1 个业务 `class`，通常只保留 1 个。
- 强制按 Ponytail 收敛：最少文件、最少函数、最少状态，服务当前工作流。
- 默认 2 到 4 个主要方法；可保留 `update_headers()`、`fetch_*()`、`run()`，必要时才加 `build_*()` 或 `write_*()`。
- 能顺序写在当前主要方法里的逻辑，不拆只调用一次的小方法。
- 不生成 `helper`、`client`、`service`、`factory`、多 `utils`、通用框架、插件体系、配置层或未来预留文件。
- 超过 1 个业务类、超过 4 个主要方法或新增独立模块时，必须有真实复用、独立认证/签名链、多子任务状态或明确业务边界。
- `self.xxx` 只保存跨方法长期状态，例如 `session`、字段映射、稳定配置。
- 输出路径、筛选条件、分页值、payload 组成参数等一次性值，作为方法参数或局部变量传递。
- 每个函数和方法上方写一句简短职责注释；页面操作/接口请求/导出链路相关注释写明“操作步骤”和“访问网址”。
- 采集请求方法体开头用三引号写参数说明：参数名、功能、类型、是否必填、格式或来源。
- 最终脚本禁止出现 `cdp_debug_url`、`webSocketDebuggerUrl`、`127.0.0.1:9222`、`/json`、`/json/version`。

## 请求与异常

- 使用 `requests.Session`。
- 每个请求显式设置 `timeout`。
- 调用 `raise_for_status()`，并检查接口自己的 `success/code/msg`。
- 只对网络超时、连接断开、`408`、`429`、`500`、`502`、`503`、`504` 做有限重试，默认不超过 3 次。
- 可能创建任务或触发导出的 POST，只有具备幂等键、可查重任务 ID 或先查已有任务时才自动重试。
- `401`、`403`、未登录、未选身份、无权限、身份失效、参数错误、验证码、MFA、风控直接抛出。
- 异步导出中的排队/生成/处理中按轮询处理，必须设置最大等待时间。
- 异常信息包含子任务、请求方法、URL、关键参数、HTTP 状态码、接口 `code/msg/success` 和响应摘要。

## 表头与文件

- 只有把接口响应重组为表格导出时，才生成 `field_mapping.json`。
- 原始下载类请求保存服务端返回的 `csv/xlsx/zip/pdf/binary`，不生成字段映射。
- 表格导出运行时读取独立 `field_mapping.json`，不要在 `.py` 中重复维护映射常量。
- 导出表头必须使用页面显示表头和顺序，不直接使用 response 字段名。
- 表格文件名按 `数据抓取时间-业务日期-业务导出表名称.<suffix>`。
- 文件名不包含店铺、商品、SKU、直播、账户等业务维度；这些字段写入文件内容。
- 重组表格输出 UTF-8 CSV；服务端原始下载文件按二进制原样保存并保留后缀。

## 大任务状态

- 多报表、多店铺、多日期分片或多子采集脚本时，使用任务状态 JSON。
- 每个子任务记录稳定 `task_key`、`input_hash`、状态、输出文件、记录数、校验摘要。
- `success` 且输出存在才跳过；`running`、`partial`、`failed`、`blocked` 下次默认重试。
- 全局认证、身份、权限或风控错误应停止整个大任务。
- 状态文件路径作为 `run()` 或 `main()` 参数传入，不挂到 `self`。

## 验证

- 核对脚本输出与页面结果：数量、ID、日期范围、样例行、表头顺序。
- 表格导出核对字段映射 JSON、UTF-8 CSV、命名格式和关键字段是否落入内容。
- 原始下载核对文件名、后缀、大小或样例打开结果。
- 说明认证参数来源：脚本生成、前置接口、用户提供。
- 运行 `scripts/validate_generated_script.py` 检查 CDP/MCP 泄漏和 Ponytail 规模。
- 写明跳过的端点、已知限制和未验证项。
